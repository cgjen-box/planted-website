## Performance Characteristics

| Operation | Latency | Caching |
|-----------|---------|---------|
| `getActiveAdapter()` | ~1ms (cached) / ~50ms (DB) | Yes |
| `shouldRollback()` | ~1ms (in-memory) | Yes |
| `rollbackAdapter()` | ~200ms (3 DB ops) | No |
| `updateAdapterStats()` | ~100ms (1 DB op) | Updates cache |
| `checkAndRollbackIfNeeded()` | ~50ms (all cached) | Yes |

## Security Considerations

1. **Version Tampering Prevention**
   - Only authorized functions can modify versions
   - Firestore security rules restrict collection access
   - Audit trail tracks all changes

2. **Rollback Authorization**
   - Manual rollbacks require admin role
   - Automatic rollbacks are system-initiated
   - All events logged with user/system attribution

3. **Alert Integrity**
   - Alerts stored in separate collection
   - Immutable once created
   - Read/unread tracking per user
